# Exploring Packaging hermes-agent

Packaging `hermes-agent==0.14.0` and its dependencies for Fedora f44 and rawhide (f45).

Reference following HTML documents for detailed buildability analysis.

- [Main buildability report](hermes-agent-buildability.html) — based on direct dependencies of hermes-agent.
- [Blocker deep dive](hermes-agent-blockers-analysis.html) — transitive dependencies of blocker dependencies that have no corresponding `f44` and rawhide builds in the build system.
- [Packaging checklist](hermes-agent-packaging-checklist.html) — trackable checkbox checklist

> [!TIP]
> These documents are generated by Pi agent assisted by skills [tkdchen/skills/fedora] and [tkdchen/skills/list-python-package-dependencies].

Builds are managed within Copr project [hermes-agent] in Fedora Copr build system.

## Benefits

- A trusted source for installing Hermes agent.
- Lower effort for users to run Hermes agent.

## Challenges

- Dynamic dependency installation during runtime.
- Large set of packages to be packaged.
- Dependencies are pinned to exact versions with `==`.
- Run hermes-agent tests under `%check` stage.

## Plan

- Define feature sets so that not all packages are installed together at once.
- Package RPM packages iteratively by feature sets.
- Disable dynamic dependency installation by default.
